## Cultural Considerations

### Formality Level
- Use formal German register (Sie-Form) for ACCORD and official documentation
- Use conversational German for UI strings and chat messages
- Maintain professional tone throughout
- Technical terms often remain in English (API, LLM, Token, Backend, etc.)

### Compound Words
- German frequently uses compound words (e.g., Einrichtungsassistent = Setup Wizard)
- Keep compounds together for natural German flow
- Break compounds with hyphens when readability requires it (e.g., KI-Konfiguration)

### Capitalization
- All nouns are capitalized in German (standard grammar)
- UI labels follow German capitalization rules
- Technical terms (API, LLM) remain uppercase

### Verb Forms
- Action verbs use infinitive form for menu items (Beobachten, Sprechen)
- Status messages use present progressive (Wird ausgeführt..., Denke nach...)
- Commands use polite imperative or infinitive

### Sie vs. Du
- Always use "Sie" (formal you) when addressing users
- Use "Ihr/Ihre" (your, formal) for possessive forms
- Maintain consistent formality across all user-facing text

### Technical Term Handling
- Keep English technical terms when they are industry standard (API, Token, Backend)
- Translate where German terms are equally common (Dienst for Service, Anbieter for Provider)
- Use English + German compound when helpful (KI-Konfiguration, API-Schlüssel)
