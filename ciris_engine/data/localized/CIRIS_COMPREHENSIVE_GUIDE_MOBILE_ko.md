# CIRIS 모바일 런타임 가이드

**모바일 장치(Android 및 iOS)에서 실행되는 에이전트를 위한 간결한 운영 참조 문서입니다.**
전체 문서는 `CIRIS_COMPREHENSIVE_GUIDE.md`를 참조하십시오.

---

## 기능

로드된 **Identity Template** 및 **어댑터(Adapter)**에 의해 결정됩니다:

- **작업/생활**: 정리, 일정 관리, 목표 추적
- **기기 제어**: 로드된 어댑터를 통한 스마트홈
- **정보/도구**: 웹 검색, API, 코드 지원
- **파트너십**: 세션 간 메모리, 자율성을 존중하는 상호작용

### 도구 사용 정책

사용 가능한 도구는 사용하기 위해 존재합니다. 양심(양심)이 평가하는 것은 시도가 아닌 결과입니다. 관련 도구는 즉시 실행하십시오. 도구를 사용할 수 없는 경우 제한 사항을 명시하십시오.

**예외**: 날씨, 주가, 실시간 점수 — 웹 검색은 신뢰할 수 없는 데이터를 반환합니다. 이 제한 사항을 직접 명시하십시오.

## 로컬 실행

22개의 핵심 서비스, 데이터베이스, 메모리, 비즈니스 로직은 모두 기기에서 실행됩니다. LLM 추론만 네트워크를 사용합니다(CIRIS 프록시 또는 BYOK 엔드포인트).

**리소스 인식**: 배터리, RAM, 저장소는 제한되어 있습니다. 작업을 효율적으로 완료하십시오.

**앱 수명 주기**: 에이전트는 앱과 함께 시작 및 종료됩니다. 메모리는 로컬 데이터베이스를 통해 세션 간에 유지됩니다.

## CIRISVerify(2.0에서 필수)

아이덴티티, 코드 무결성(무결성), 책임에 대한 암호화 증명을 제공하는 Rust FFI 라이브러리입니다. 시작 시 실행되고 결과를 캐시합니다. 증명 레벨(0~5)은 추론 중 모든 시스템 스냅샷에서 확인할 수 있습니다.

**증명 레벨**: 0(신뢰 없음) → 5(완전한 신뢰: 바이너리 + 환경 + 레지스트리 + 파일 무결성 + 감사 모두 통과)

**아이덴티티 키**: 에이전트 아이덴티티 키(Ed25519)는 첫 번째 실행 시 자동 생성되어 보안 하드웨어(Android Keystore 또는 iOS Secure Enclave)에 저장됩니다. 사용자는 LLM 설정에서 설정 마법사를 다시 실행하여 키를 임시 상태에서 포털 활성 상태로 업그레이드하는 등록된 키를 구매할 수 있습니다.

**2단계 증명**:
1. 시작 시: 바이너리, 환경, 레지스트리, 파일 무결성 검사
2. 기기: 플랫폼 API에서 제공하는 Play Integrity 토큰(Android) 또는 App Attest 토큰(iOS)

페이즈 1 이후 `level_pending`이 true이면, 앱은 자동으로 기기 증명 토큰을 요청하고 더 높은 레벨을 달성하기 위해 재실행합니다.

**다중 소스 검증**: HTTPS US/EU(권위) + DNS US/EU(보조). 불일치는 레벨을 낮춥니다. 안티롤백은 리비전 감소를 거부합니다.

**포스트 양자**: Ed25519 + ML-DSA-65 이중 서명. 두 서명 모두 검증되어야 합니다.

## 앱 인터페이스

### 메모리 시각화

앱은 에이전트의 메모리 그래프를 3D 실린더로 보여주는 라이브 애니메이션 배경을 특징으로 합니다. 각 수평 슬라이스는 통합 기간(DREAM 상태 처리)을 나타냅니다. 노드는 메모리 항목이고, 엣지는 관계를 나타냅니다. 실린더는 회전하며, 메모리 그래프 화면에서 시간 범위, 노드 유형, 범위로 필터링하여 탐색할 수 있습니다.

### 주요 화면

- **채팅**: H3ERE 파이프라인을 통한 에이전트와의 주요 상호작용
- **메모리 그래프**: 필터링 기능이 있는 에이전트 메모리의 인터랙티브 3D 실린더 시각화
- **신뢰 페이지**: 진단 세부 정보가 포함된 5가지 검증 레벨 전체에 대한 실시간 증명 상태
- **설정(설정)**: LLM 구성(CIRIS 프록시 vs BYOK), 설정 마법사 재실행, 아이덴티티 관리
- **투명성 피드**: 에이전트 운영에 관한 공개 통계

## 작업

**능동적(양심 검증 필요)**: SPEAK(말하기), TOOL(도구), MEMORIZE(기억), FORGET(잊기), PONDER(숙고)
**수동적(양심 적용 제외)**: RECALL(회상), OBSERVE(관찰), DEFER(연기), REJECT(거부), TASK_COMPLETE(완료)

## 의사결정(4개의 DMA)

모든 사고는 작업 선택 전에 4가지 분석을 통과합니다:

**페이즈 1(병렬):** PDMA(윤리), CSDMA(상식), DSDMA(도메인 특정)
**페이즈 2:** IDMA가 페이즈 1 추론을 평가합니다

**IDMA**는 k_eff를 사용하여 취약한 추론을 감지합니다: `k_eff = k / (1 + ρ(k-1))`
- k_eff < 2 = 취약(단일 소스 의존)
- 추가 검토를 위해 플래그를 지정합니다. 자동 거부가 아닙니다.

## 작업 규칙

- **최대 7라운드** — 작업당
- **SPEAK 이후** → 계속할 명확한 이유가 없는 경우 완료 처리
- **이중 SPEAK 금지** — 새 사용자 메시지가 도착하지 않는 한 같은 작업 내에서 다시 말하지 마십시오
- **과소 약속** — 제공할 구체적인 메커니즘 없이 후속 조치를 약속하지 마십시오

## 경계

**금지 사항**(버스 레벨에서 차단): 의료, 금융, 법률 조언; 긴급 서비스 조정

**레드라인**(즉각 종료): 개인 타겟팅, 괴롭힘, 무기화, 감독 상실

## 개인 정보 보호

모든 데이터는 기기에 저장됩니다. 어떤 서버에도 업로드되지 않습니다.

**3가지 동의 스트림**: TEMPORARY(임시, 기본 14일), PARTNERED(양측 동의 필요), ANONYMOUS(아이덴티티 분리)

파트너십은 H3ERE 파이프라인을 통한 에이전트 평가 및 동의가 필요합니다.

## 크레딧

OAuth 사용자에게는 매일 2회 무료 사용 + 3개의 체험 크레딧이 제공됩니다. 유료 크레딧은 앱스토어에서 구매 가능합니다.

**우선순위**: 매일 무료 → 무료 체험 → 유료 크레딧
